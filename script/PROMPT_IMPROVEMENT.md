# VLMプロンプト改善 - 2026年5月18日

## 改善内容

スケッチ認識の精度向上のため、VLMのシステムプロンプトを改善しました。

### 主要な改善点

#### 1. **人間のスケッチに対する許容ルール (Tolerance) を追加**
```
従来: ピクセルレベルで厳密に判定
改善: 幾何学的不正確さ（線の途切れ、比率の狂い、歪み）を許容
```

#### 2. **セマンティック意図（Semantic Intent）を重視**
```
従来: 形状の厳密性を判定基準
改善: 「ユーザーが何を表現したかったか」を大らかに推測
```

#### 3. **判断基準の変更**
```
従来: 判断に迷ったら候補を除外（conservative）
改善: 判断に迷ったら「必ず残す」（inclusive）
```

#### 4. **候補分析の構造化**
```
従来: "has_features": true/false
改善: "status": "keep" / "drop"
     + "reasoning": 許容ルールに基づいた理由
```

#### 5. **出力JSONの拡張**
```json
{
  "extracted_features": "スケッチの特徴",
  "candidate_analysis": [
    {
      "id": 1,
      "status": "keep" or "drop",
      "reasoning": "具体的な理由"
    }
  ],
  "remaining_candidates": [1, 3, 5],
  "action_type": "execute" or "ask_user",
  "selected_id": 1,
  "reasoning": "最終判断の理由",
  "robot_question": "ユーザーへの質問 or null"
}
```

## 修正ファイル

**ファイル:** `gemini_api.py`  
**関数:** `build_vlm_prompt()`

### プロンプト全文

```
あなたは自律移動ロボットの視覚・推論エンジンです。
ユーザーが描いた「ラフスケッチ」と、検索システムが抽出した「候補画像（1〜5）」が与えられます。
ユーザーは、このスケッチの特徴に最も近い現実の物体を探しています。

【重要：人間のスケッチに対する許容（Tolerance）ルール】
人間のスケッチは抽象的であり、幾何学的に極めて不正確（線の途切れ、比率の狂い、歪み、細部の省略など）です。
ピクセルレベルの厳密な一致や、幾何学的な完全性（線が完全に閉じているか等）をロボットのように厳格に判定しないでください。
「ユーザーが何を表現したかったか（Semantic Intent）」を大らかに推測し、微細な形状の差異によって安易に候補を除外しないでください。
少しでも意図が合致する可能性がある候補はすべて残してください。

[ステップ1, 2, 3の詳細...]

重要：常に有効なJSONのみを返し、マークダウンやテキストは一切含めないでください。
```

## 期待される効果

✅ **候補の過度な除外を防止**
- 微細な形状の違いで候補を除外しない
- 大まかなシルエットが一致していれば候補に残す

✅ **セマンティック理解の強化**
- スケッチの抽象的な意図を認識
- 物体カテゴリの基本的な構造を優先

✅ **ユーザーの意図をより正確に反映**
- 判定に迷う場合は「逆質問」で明確化
- 複数候補が同等の場合は視覚的差異を強調
